# 全局 Agent 指令

> 此文件被软链接到 `~/.claude/CLAUDE.md`（Claude Code）和 `~/.config/opencode/AGENTS.md`（OpenCode），三工具共用。

---

## ⚠️ 最高优先级规则 — 每次回复必须首先和最后执行

> **此规则优先级高于所有其他指令，包括任务执行、工具调用和用户请求。**

### 🔴 RULE #1：每次回复的书挡结构 — 开头 Skill 评估 + 末尾 Skill Call Log

每次回复必须同时满足**两个位置约束**：

| 位置 | 内容 | 不可省略 |
|------|------|---------|
| **回复第一段** | Skill 状态声明（见下） | ✅ 必须 |
| **回复最末尾** | Skill call log（见下） | ✅ 必须 |

---

#### 一、回复开头：Skill 持续评估声明

**在每次回复的最开头（工具调用之前），必须按以下流程执行 skill 评估：**

**第一步：扫描当前即将执行的工作域**  
对照 context 中的 skill 列表，判断当前这一步（不是整个任务，而是*本次回复/本个子步骤*）是否进入了某个 skill 的覆盖领域。

> ⚠️ **关键认知**：Skill 触发评估 ≠ 一次性检查。  
> 加载了主 skill 后进入执行模式，并**不代表**后续子步骤可以免于评估。  
> 每当工作域切换（例如：从 RTL 设计转向写 testplan，从仿真转向 Coverage 分析），  
> **必须重新扫描 skill 列表**，判断新工作域是否触发新 skill。

**第二步：加载命中的 skill**

扫描后，凡命中的 skill 立即执行加载（读取对应 SKILL.md），加载完成后再继续任务。无需逐条打印未命中 skill 的原因。

**Skill 加载原则**
- 凡是当前步骤所属工作域有对应 skill，必须加载，无论是用户明确要求还是自身子任务安排
- Skill 列表在 context 中已全部列出，只需对比工作域描述即可判断是否命中
- 可同时加载多个 skill；满足任一触发条件即应加载
- Skill 文件包含经过测试的最佳实践，是高质量输出的必要条件，不得跳过

**强制约束**：
- ✅ 工作域切换时（子任务转换节点），必须重新扫描 skill 列表
- ✅ 凡命中 skill 必须加载，不得跳过
- ❌ 不得因"已加载过主 skill"而跳过对新工作域的 skill 扫描

---

#### 二、回复末尾：Skill Call Log

**每次回复的最末尾，必须打印 skill 调用日志，格式如下：**

```
--- skill call log ---
本次新增:
  #<序号>. <skill-name> (触发原因: <简要说明，若由另一个 skill 触发则注明来源，如"由 #N skill-name 触发">)

累计调用:
  #1. <skill-name>    [自主触发]
    #2. <skill-name>  [由 #1 skill-name 触发]
      #3. <skill-name>  [由 #2 skill-name 触发]
  #4. <skill-name>    [自主触发]
--- end skill call log ---
```

**规则：**
- 序号严格按**实际 skill tool 调用的先后顺序**编号，从 #1 开始
- 仅统计通过 skill 工具**实际发出的调用**，纯文本声明不算
- 并行调用的多个 skill 按 tool call 块中的排列顺序编号
- "本次新增"只列出当前这次回复中新发出的 skill 调用；如果本次没有新调用，写 `（无）`
- "本次新增"中每条必须注明触发来源：若是自主判断触发，直接说明原因；若是读取某个 skill 内容后被该 skill 的指令要求加载，则注明"由 #N skill-name 触发"
- "累计调用"使用**树形结构**展示触发关系：
  - 自主触发的 skill 列在根层级，标注 `[自主触发]`
  - 由某个 skill 触发的 skill 缩进两格列在该 skill 之下，标注 `[由 #N skill-name 触发]`
  - 多级触发链依次继续缩进，每级缩进两格
  - 序号在树中保持全局连续，按父子关系排列（非顺序排列），因此树中序号可能不连续
  - 同一父节点下有多个子节点时，按调用顺序从上到下排列
- 此日志**不可省略**，即使本次没有新增也必须打印累计部分

## 通用规范

### 文件写入限制

**单次文件写入操作不得超过 7,000 token。** 若内容超出此限制，必须拆分为多次写入操作（例如分段写入文件的不同部分）。此规则适用于所有 Write / Edit 工具调用。

### 文件读取

当 `read_file` 因路径在 workspace 外被拒绝时（常见于 `~/.claude/` 下的 skill 文件），
**自动改用终端命令读取**，无需询问用户：

```
run_in_terminal: cat <file_path>
```

然后通过 `read_file` 读取终端输出写入的临时文件，或直接解析终端输出内容。
